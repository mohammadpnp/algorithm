# نقشه راه آمادگی لایو کدینگ الگوریتمی (Go)

این مجموعه برای آمادگی مصاحبه بکند گولنگ طراحی شده و روی الگوهای پرتکرار سوالات الگوریتمی تمرکز دارد.

## ساختار استفاده

برای هر تایپ سوال یک پوشه دارید که دقیقاً 3 فایل دارد:

1. `lesson.md`: درسنامه کامل، الگوی ذهنی، فوت و فن‌ها، اشتباهات رایج
2. `solved-example.md`: حل کامل یک سوال نمونه با کد Go
3. `practice.md`: یک سوال مشابه برای تمرین شخصی

## تایپ‌های سوالات در این پک

1. `01_two_pointers_sliding_window`: دو اشاره‌گر و پنجره لغزان
2. `02_hashmap_set_frequency`: هش‌مپ، ست، شمارش فراوانی
3. `03_prefix_sum_difference`: پیشوند جمع و آرایه اختلاف
4. `04_binary_search`: جستجوی دودویی روی آرایه و پاسخ
5. `05_stack_monotonic_stack`: پشته و پشته یکنواخت
6. `06_heap_priority_queue`: هیپ و صف اولویت
7. `07_linked_list`: لیست پیوندی و الگوهای کلاسیک
8. `08_tree_dfs_bfs`: درخت، DFS، BFS
9. `09_graph_bfs_dfs_toposort`: گراف، پیمایش، توپولوژیکال سورت
10. `10_dynamic_programming`: برنامه‌نویسی پویا
11. `11_greedy_intervals`: حریصانه و بازه‌ها
12. `12_backtracking_bitmask`: بک‌ترکینگ و الگوهای جستجوی کامل

## فوت و فن‌های کلیدی مصاحبه لایو کدینگ

1. قبل از کدنویسی مسئله را بازنویسی کن: ورودی، خروجی، قیدها، edge case.
2. اول brute force را بگو، بعد بهینه‌سازی را معرفی کن.
3. صریح بگو چرا یک ساختار داده خاص را انتخاب کردی.
4. حین کدنویسی پیچیدگی زمانی و حافظه را با صدای بلند تحلیل کن.
5. ابتدا امضای تابع و تست ذهنی نمونه‌های کوچک را بنویس.
6. روی مرزها حساس باش: آرایه خالی، `k=0`، تکرار، عدد منفی، overflow.
7. در Go مراقب مقدار اولیه map/slice و nil pointer باش.
8. برای کدهای stateful از متغیرهای کوتاه اما معنادار استفاده کن (`left`, `right`, `freq`).
9. اگر گیر کردی: مسئله را به الگوهای شناخته‌شده نگاشت کن (window, dp, graph, ...).
10. پایان کار یک dry-run خیلی سریع روی ورودی نمونه انجام بده.

## ترتیب تمرین پیشنهادی

1. دو اشاره‌گر + هش‌مپ + پیشوند جمع
2. باینری سرچ + استک یکنواخت + هیپ
3. لیست پیوندی + درخت + گراف
4. DP + greedy + backtracking

اگر روزی 2 تا 3 مسئله از هر تایپ تمرین کنی، طی چند هفته جهش خیلی خوبی می‌گیری.
